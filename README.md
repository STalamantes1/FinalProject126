# Final Project: Programming Project with AI-Assisted Revision

## Overview

For your final project, you will create a Python program of your choice that demonstrates the programming skills we have practiced throughout the semester.

This project should be more detailed and complete than a regular lab assignment. You will first create a working version of your program on your own. After that, you may use Claude AI to help improve, polish, debug, or expand your project.

The goal is not to have AI create the entire project for you. The goal is to show that you can build a working program, use AI as a programming assistant, and understand the final product.

Your final project will be submitted through GitHub. You will submit a public GitHub repository link on Canvas.

---

## Project Goals

By the end of this project, you should be able to:

- Create a complete Python program of your choice
- Organize code using functions and/or classes
- Use programming concepts covered throughout the semester
- Use a Python library or module to complete part of the project
- Store or organize data in a meaningful way
- Use GitHub to submit a complete programming project
- Use AI responsibly as a development tool
- Explain the difference between your original code and your AI-assisted final version

---

## Project Requirements

Your project must:

1. Be written in Python unless another language is approved by the instructor.

2. Be a program of your choice.

3. Include user interaction, such as:
   - menus
   - user input
   - choices
   - displayed results
   - repeated actions

4. Use functions to organize your code.

5. Use at least one Python library or module.
   - Built-in Python libraries are preferred.
   - Examples: `random`, `datetime`, `os`, `json`, `csv`, `math`, `tkinter`
   - Avoid projects that require multiple `pip install` commands unless approved by the instructor.

6. Store or organize data in a meaningful way.

   Examples include:

   - lists
   - dictionaries
   - files
   - JSON files
   - CSV files
   - classes/objects

7. Demonstrate multiple programming concepts covered in class.

   Examples include:

   - variables
   - input/output
   - conditionals
   - loops
   - functions
   - lists
   - dictionaries
   - file input/output
   - error checking
   - classes and objects
   - modules
   - basic testing/debugging

8. Be understandable to another person who opens your GitHub repository.

9. Run without missing files or major errors.

---

## AI Assistance Policy

You are allowed to use Claude AI to help with your final project.

However, you must first create your own starter version of the program before using Claude.

Your starter code does not need to be perfect, but it should show your own original attempt at the project. It should include the basic idea, structure, and functionality of your program.

After your starter code is created, you may use Claude to help improve the project.

Claude may help you:

- debug errors
- clean up your code
- improve formatting
- add comments
- improve function organization
- suggest better variable names
- improve the user menu
- add small features
- improve the README
- help write or improve the `specs.md` file

Claude should not replace your understanding of the project.

You must be able to explain:

- what your program does
- how your code works
- what your starter code did
- what changed in the final version
- how Claude helped
- what parts of the code you understand and modified
